# connection_points

object containing all the points that a path can connect to in this node the points here only descript the position the paths should connect to... and it's updated by the node core when the node change other classes may use this object to know the position of connection and display any kind elements on the same position in the dom .. however this object doesn't have any elements on the dom .. just a represntaion of the connection position.... The format of this object ISN"T STANDARDIZED yet .. will be standardized when i have a genral idea of all path requirement currntly the default node_core fills in the object with the format suitable for the default path core. the object format is important as the path classes interact with it based on this format the current format allows for other point to be added in case needed ... different path creation algorithms (implemented in path_core extender classes) may exploit this object
